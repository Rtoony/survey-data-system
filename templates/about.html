System Philosophy & Vision

The Companion Tool Concept

ACAD-GIS is not a replacement for traditional CAD software like AutoCAD or Civil3D. Instead, it serves as an intelligent companion that works alongside your existing CAD tools.

While CAD software excels at nuanced drafting and precise geometric creation, ACAD-GIS shines in areas where traditional CAD struggles:

  • Project Status Tracking: Real-time visibility into project health, completeness, and compliance
  • Performance Analysis: Metrics, statistics, and quality scoring across your entire project portfolio
  • Semantic Understanding: AI-driven interpretation of CAD data for intelligent search and analysis
  • Specialized Management Tools: Domain-specific interfaces for gravity pipes, pressure networks, BMPs, survey data, and more
  • Automated Compliance: Continuous validation against standards, detecting inconsistencies automatically
  • Multi-Project Intelligence: Cross-project analysis, pattern detection, and reusable standards libraries

Why Database-Centric vs. File-Based CAD?

Traditional CAD workflows rely on individual DWG/DXF files as the source of truth. This creates fundamental limitations:

❌ File-Based Limitations
  • • Data locked in binary files
  • • No semantic relationships
  • • Manual cross-file searches
  • • Version conflicts
  • • Limited AI understanding
  • • No automated compliance
✓ Database-Centric Benefits
  • • Single source of truth
  • • Explicit relationships (GraphRAG)
  • • Instant semantic search
  • • Built-in version control
  • • AI-optimized structure
  • • Real-time compliance tracking

The workflow: Import DXF files into the database, manage and analyze your data using ACAD-GIS specialized tools, then export back to DXF when you need to make geometric edits in CAD software. This creates a bidirectional sync that combines the best of both worlds.

AI-First Architecture

Every design decision in ACAD-GIS is optimized for machine understanding and AI capabilities:

  • Vector Embeddings: 1536-dimension semantic representations of all CAD entities
  • GraphRAG: Explicit graph edges for spatial, engineering, and semantic relationships enabling multi-hop queries
  • Hybrid Search: Combines vector similarity, full-text search, and spatial queries
  • Quality Scoring: Automated quality assessment based on completeness, relationships, and embeddings
  • Truth-Driven: All vocabulary, standards, and configurations stored in database, never hardcoded

Core Architecture

ACAD-GIS is built on five interconnected architectural components that work together to create an intelligent, self-updating system. Each component serves a specific purpose while contributing to the overall truth-driven architecture.

1. CAD Layer Vocabulary
Purpose: Defines the language of CAD layer naming

Format:

DISCIPLINE-CATEGORY-TYPE-ATTRIBUTES-PHASE-GEOMETRY

Example:

CIV-UTIL-STORM-12IN-NEW-LN

Components:

  • • CIV = Civil discipline
  • • UTIL = Utilities category
  • • STORM = Storm drain type
  • • 12IN = 12-inch diameter
  • • NEW = New construction
  • • LN = Line geometry
Database Tables:
discipline_codes, category_codes, object_type_codes, phase_codes, geometry_codes, attribute_codes
2. Entity Registry
Purpose: The "switchboard" connecting layer names to database tables

Mappings:

  • • STORM → utility_lines table
  • • MH → utility_structures table
  • • SURVEY → survey_points table
  • • BLDG → buildings table
  • • ROAD → roadways table

What it enables:

  • ✓ Automatic DXF import routing
  • ✓ Correct database table selection
  • ✓ Entity type identification
  • ✓ Specialized tool discovery
Database Tables:
standards_entities (dynamic), built-in registry (static fallback)
3. Specialized Tools Registry
Purpose: Links object types to management tools

Three-Dimensional Filtering:

Object Type → Attribute → Database Table

Examples:

  • • STORM → Gravity Pipe Manager
  • • WATER → Pressure Pipe Manager
  • • BIOR → BMP Manager
  • • SURVEY → Survey Point Manager

Surgical Precision:

  • BASIN+STORAGE → volume calcs
  • BASIN+TREATMENT → water quality
Database Tables:
specialized_tools, layer_objects, layer_object_tools
4. Reference Data Hub
Purpose: Project-agnostic reference data management

9 Reference Data Types:

  • 1. Attribute Codes
  • 2. CAD Standards
  • 3. Clients
  • 4. Municipalities
  • 5. Coordinate Systems
  • 6. Survey Point Descriptions
  • 7. Naming Templates
  • 8. Tool-Object Mappings
  • 9. Specialized Tools Registry
Key Feature:
Global library reusable across all projects
5. Standards Mapping Framework
Purpose: Bidirectional DXF ↔ Database translation

11-Table Schema:

  • • keynote_blocks
  • • keynote_details
  • • hatch_materials
  • • detail_materials
  • • block_specifications
  • • cross_references
  • • project_keynote_blocks
  • • project_keynote_details
  • • project_hatch_materials
  • • project_detail_materials
  • • project_block_specs

5 Name Mappers:

  • ✓ Block-to-Keynote Mapper
  • ✓ Detail-to-Keynote Mapper
  • ✓ Hatch-to-Material Mapper
  • ✓ Detail-to-Material Mapper
  • ✓ Block-to-Spec Mapper
Purpose:
Translate CAD element names to semantic database attributes

Bidirectional Sync Workflow

The power of ACAD-GIS lies in this bidirectional workflow. You can move freely between database-driven analysis and traditional CAD drafting, with changes synchronized in both directions. The change detection system identifies modified entities, ensuring no data loss during round-trip operations.

Key Differentiators

ACAD-GIS introduces capabilities that are impossible in traditional file-based CAD workflows. These features represent fundamental architectural advantages of the database-centric approach.

GraphRAG & Semantic Search

What it is:

Graph Retrieval-Augmented Generation combines explicit relationship edges with vector embeddings to enable multi-hop reasoning across your CAD data.

Example query:

"Find all storm drains that connect to structures within 500 feet of buildings on Main Street"

This type of semantic, relationship-aware query is impossible in traditional CAD systems.

Automated Compliance Tracking

What it is:

Project Relationship Sets continuously monitor dependencies between CAD elements, specifications, details, notes, and materials. Violations are detected automatically.

Example detection:

  • ✗ Block "INLET-TYPE-B" used but Detail "D-INLET-B" not in project
  • ✗ Hatch pattern changed but Material spec outdated
  • ✗ Required note missing from sheet set
Multi-Project Intelligence

What it is:

Query and analyze data across your entire project portfolio simultaneously. Identify patterns, reuse successful solutions, and learn from historical projects.

Examples:

  • • "Which inlet type is most used across all projects?"
  • • "Show me all bioretention designs from the last 3 years"
  • • "What pipe materials work best in coastal areas?"
Built-in Version Control

What it is:

Every change to CAD data is tracked with timestamps, user attribution, and full audit trails. No more manual file versioning or "FINAL_v3_REVISED_FINAL.dwg".

Features:

  • ✓ Change detection on DXF import
  • ✓ Entity modification history
  • ✓ Rollback capabilities
  • ✓ Diff visualization
Truth-Driven Architecture

What it is:

All vocabulary, standards, and tool configurations live in the database. Changes propagate automatically across the entire system without code modifications.

Benefits:

  • ✓ Add new object types without coding
  • ✓ Client-specific standards customization
  • ✓ Dynamic UI updates
  • ✓ Instant vocabulary refresh
Quality-Driven Workflows

What it is:

Every entity has a quality score based on completeness, embedding coverage, relationship health, and metadata richness. Surface data quality issues automatically.

Scoring factors:

  • • Required attributes present (30%)
  • • Vector embedding exists (25%)
  • • Graph relationships defined (25%)
  • • Metadata completeness (20%)
