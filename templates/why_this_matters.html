Unconventional Ways to Analyze Your Data

Things you could never do with traditional CAD workflows

Semantic Relationship Discovery

Query: "Show me everything conceptually related to 'drainage'"

Result: AI finds catch basins, inlets, pipes, manholes, BMP facilities, treatment devices, even if they use different naming conventions—because embeddings understand meaning, not just keywords.

GraphRAG Multi-Hop Queries

Query: "What materials are required for details referenced by keynotes in Building A?"

Result: GraphRAG follows: Drawing → Keynotes → Details → Materials through explicit relationship edges, showing you the complete supply chain for that building.

Cross-Project Spatial Analysis

Query: "Find all fire hydrants within 300 feet of schools across every project in the database"

Result: PostGIS spatial query returns results in 2 seconds, complete with distances and project names—impossible with file-based CAD.

Standards Adoption Trends

Analysis: Drawing Usage Dashboard shows which layers are actually used vs. just defined in templates

Insight: Discover that 40% of your layer standards are never used, and 3 non-standard layers appear in 80% of projects—data-driven standards refinement.

Automated Quality Scoring

Analysis: Quality Dashboard calculates health metrics for every entity

Metrics: Completeness score (12 factors), embedding coverage (semantic understanding), relationship density (how connected), standards compliance percentage.

Network Topology Validation

Analysis: Gravity Network Manager checks structural rules

Validation: "This manhole has 1 inlet pipe and 3 outlet pipes (unusual—verify design)" or "Pipe slope 0.002 is below minimum 0.004 for 8" diameter"—engineering intelligence built-in.

Proximity-Based Compliance

Query: "Find all ADA ramps not within 5 feet of a parking space"

Result: Spatial validation catches accessibility compliance issues before construction—combine CAD + GIS + compliance rules in one query.

Layer Usage Heatmaps

Analysis: "Which layers have the most entities across all projects?"

Result: Discover UTIL-STORM-PIPE appears in 95% of civil drawings with 12,000+ entities, while LAND-TREE-DECID has 3 entities total—optimize your standards library.

Bidirectional Name Mapping

Capability: Import DXF with blocks named "CB-TYPE1" → maps to database standard "CATCH_BASIN_TYPE_A"

Export: Reverses mapping back to "CB-TYPE1" for client compatibility—5 mapping managers handle blocks, details, hatches, materials, notes.

Coordinate System Fusion

Capability: Store CAD data in SRID 0 (local coordinates) AND GIS data in SRID 2226 (California State Plane)

Transform: "Convert this CAD line to GIS coordinates" or "Overlay aerial imagery on CAD drawing"—seamless transformation via PostGIS ST_Transform().

Project Relationship Intelligence

Query: "Show me all details that reference Material X but that material isn't attached to this project"

Result: Project Relationship Manager tracks 6 relationship types—catches missing specifications before they become RFIs during construction.

Complete Audit Trails

Tracking: Every import, export, edit, deletion recorded with timestamps and user IDs

Query: "Show me all changes to Project XYZ between March 1-15" or "Who exported Drawing ABC on 3/10?"—forensic-level data governance.

Real-World Workflows

How engineers actually use the system end-to-end

Storm Drain Design Project

Workflow:

  1. Create new project in Project Manager, attach client/municipality/coordinate system
  2. Import site topo DXF via Batch CAD Import → system validates against Standards Library
  3. Use Gravity Network Manager to design storm pipes with slope/invert intelligence
  4. Visualize design in Entity Viewer (no CAD software needed) and Map Viewer for GIS overlay
  5. AI Toolkit generates embeddings and suggests related BMPs based on drainage area
  6. Quality Dashboard checks design completeness (all pipes connected, slopes valid, materials specified)
  7. Project Relationship Manager ensures keynotes reference correct details and materials
  8. Export clean DXF with perfect round-trip fidelity for client review
  9. Compliance Tab shows project meets 95% of company standards

Result: 4-hour design process with automated QC, no manual standards checking, complete audit trail.

Multi-Site Asset Inventory

Challenge: University needs inventory of all ADA ramps across 50 building plans to verify compliance.

Traditional: Open 50 DWG files, manually find ramps in each, measure distances to parking, create spreadsheet (2 days).

With Platform:

  1. Spatial query: SELECT * FROM block_inserts WHERE block_name LIKE '%ADA_RAMP%' AND project_id IN (campus_projects)
  2. Results in 5 seconds: 127 ramps with coordinates, project names, insertion dates
  3. Use Map Viewer to visualize all ramps on aerial imagery
  4. Proximity analysis: Find ramps NOT within 5 feet of parking spaces → identifies 8 compliance issues
  5. Export report to Excel with coordinates, photos (from attributes), compliance status

Result: 30-minute analysis vs. 2-day manual effort. Issues caught before accessibility audit.

Standards Refinement Initiative

Goal: Engineering firm wants to update CAD standards based on actual usage, not assumptions.

Analysis:

  1. Drawing Usage Dashboard analyzes 200 projects over 2 years
  2. Discovery: 60% of defined layers never used, 5 non-standard layers appear in 90% of drawings
  3. DXF Name Translator shows 30 different DXF names mapping to same database block (inconsistent naming)
  4. Quality Dashboard reveals projects with <80% completeness scores correlate with RFIs during construction
  5. GraphRAG analysis: "Which keynotes reference obsolete details?" → finds 45 outdated references

Actions:

  1. Retire 40 unused layers via Layer Manager
  2. Promote 5 commonly-used non-standard layers to official standards
  3. Consolidate block name mappings to 1 canonical name per block
  4. Update keynote-detail relationships in Project Relationship Manager
  5. Export new standards guide via Standards Export (PDF + Excel for team training)

Result: Data-driven standards update reduces layer count by 40%, improves consistency, decreases RFIs by 25%.

Emergency Infrastructure Query

Scenario: City needs to locate all water valves within 500 feet of a gas leak for emergency shutoff.

Traditional: Call 3 different engineering firms who designed various districts, wait for them to open files, manually search (hours/days).

With Platform:

  1. Spatial query: Find all blocks with 'VALVE' in name within 500 feet of POINT(lat, lon)
  2. Results in 3 seconds: 12 valves with exact coordinates, project names, installation dates
  3. Pressure Network Manager shows which pipes each valve controls
  4. Map Viewer displays valves on aerial imagery with measurement tools
  5. Export coordinates to field crews' GPS devices

Result: 5-minute response vs. multi-hour emergency coordination. Lives potentially saved.
